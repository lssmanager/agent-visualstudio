Overview debe ser un control plane real, no un mock de tarjetas.

Este mockup propone un panel tipo Flowise para OpenClaw Studio donde la fuente de verdad es OpenClaw: agencias, departamentos, workspaces, agentes, subagentes, runs, costos, tokens, modelos, caché, tool calls, sesiones, handoffs, webhooks, flows, conexiones, interconexiones, diagnósticos y errores. La vista está pensada como dashboard redistribuible por widgets, con filtros por entidad, costo, modelo, estado, actividad y salud de runtime.

OpenClaw source contract
Este mockup asume que el overview consume datos reales de OpenClaw mediante una capa backend de agregación. No debe mostrar datos hardcodeados en producción; cada widget debe leer métricas reales, soportar filtros globales y exponer drill-down hacia Workspace Studio, Agency Topology, Observability y Replay.
